    Ubuntu下搭建Minecraft Forge服务端【踩坑记录】

    前言

    没有前言，就是闲着没事想整一个服务器。

    准备

    安装Java无头包

    sudo apt install openjdk-8-jre-headless

    通过 -version验证

    java -version

    我的输出：

    openjdk version "1.8.0_252" OpenJDK Runtime Environment (build 1.8.0_252-8u252-b09-1~18.04-b09) OpenJDK 64-Bit Server VM (build 25.252-b09, mixed mode)

    安装Forge服务端

    forger官网

    这一步有各种不同的版本，有的通过wget下载forge的install版本，然后通过指令在服务器安装，有的直接下载universal版本和对应的server.jar，再上传自己的libraries库。对于前者，国内真的几乎没办法安装成功，对于后者，基本上库和核心不匹配。

    我的解决办法就是下载forge的install版本到本机，然后双击jar，挑个空白的文件夹，将forge的server版本安装到自己的电脑，理论上，你将会得到以下文件：

    然后再通过Xftp或者其他的软件，将安装好的forge上传到你的服务器。

    在服务器上运行

    其实除了forge的就没啥坑了，现在只需要在服务器上运行即可

    java -Xms1024m -Xmx2048m -jar forge-1.12.2-14.23.5.2854.jar nogui

    -Xms 最小内存

    -Xmx 最大内存

    这两个根据自己服务器的配置进行调整

    第一遍运行会生成一个Eula.txt，并需要你同意，这些都和其他的教程，没啥区别了。

    添加mod只需要向本文件夹下的mods文件夹（即与libraries文件同级）添加即可。

    然后再次运行上述指令。

    至于用screen指令让其在后台运行的部分，没什么坑，所以不再赘述
